Human68k スーパバイザスタック消費量の表示 (stackfree)
=====================================================

## 概要

Human68k のスーパバイザスタック領域はHuman68kがロードされるアドレス$006800の直前に置かれていて、DOSコールやIOCSコールの実行によってスーパバイザモードに入った時や、実行中に発生する割り込み処理の際に用いられます。
このプログラムは、こうした処理でスーパバイザスタックがどの程度使用されているのかを計測して表示します。

## 使い方

計測は、まずスーパバイザスタック領域を一定の値($aa)で埋めて、それがどこまで書き換えられたかを調べることで行います。

stackfreeコマンドにオプション(なんでも良いです)を付けて起動すると、
```
スーパバイザスタック消費量計測を開始します
```
と表示されるので、その後オプションを付けずにstackfreeコマンドを実行します。
すると、以下のようにスタック領域のアドレス範囲とスタックポインタ、現時点での最大消費量を表示します。
```
スーパバイザスタック:   0x00003800 - 0x00008372 (19314bytes)
現在のスタックポインタ: 0x00008364 (used 14bytes, free 19300bytes)
最大スタック消費量:     0x0000812c (used 582bytes, free 18732bytes)
```

※ Human68kの起動時の初期化処理は一度しか実行されないため、初期化が終了した後はその部分がスーパバイザスタック領域に転用されます(v3.02の場合 $006800-$008371の範囲)。
